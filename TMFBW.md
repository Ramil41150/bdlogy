# Блокировки
В РФ ввели интернет-цензуру, в связи с этим провайдеров обязали устанавливать "чёрные ящики".
"Ящики" называются ТСПУ, что значит "Технические Средства Противодействия Угрозам".

## ТСПУ
ТСПУ является комплексом средств для блокировки, который расположен у провайдера. Одним из средств является DPI (Deep Packet Inspection, Глубокая Проверка Пакета). DPI можно обойти с помощью ByeDPI.

 ### Этапы проверки пакета

 1. Пакет отправляется по маршруту

 2. Пакет проходит на первую часть анализа — байпас. Байпас отправит пакет на балансировщик или ТСПУ (зависит от маштаба сети оператора). (По сути просто отвечает за перехват).

 3. Байпас отправляет пакет на балансировщик. Балансировщик проверит, какие фильтры сейчас свободны и отправит пакет на свободный фильтр (в отличие от вида ТСПУ, он может самостоятельно выполнять анализ на L3, L4 модели OSI).

 4. Фильтр — проверка пакетов на приметы, которые указаны в правилах ТСПУ (вроде, L7).

## DPI

> [!TIP]
> *Не путайте ByeDPI и DPI*. Если вы хотите сократить, то пишите "бд" или "bd"

 DPI фильтрует ваш трафик, если он обнаружит определённый трафик, который по сценарию запрещён, то будет применён определённый вид блокировки.

 У DPI есть несколько вариаций анализа:
 
 #### Обычный анализ трафика.
   При отправке клиентом на сервер сообщения о шифрование трафика (TLS Handshake). DPI перехватит его и проанализирует трафик.
   При проверке будет проверяться SNI (Индикатор Имени Сервера).
   Для примера: SNI = youtube.com — блокировка соединения по сценарию, SNI = vk.com - отправка далее.

  ### Какими методами можно отразить трафик?

  У DPI есть несколько вариантов действий

  - RST (от 'reset', сброс) — генерируется RST пакет, который мгновенно сорвёт ваше соединение с сервером (для Chrome ERR_CONNECTION_RESET)

  - FIN ('final', конец) — генерируется FIN пакет, который указывает на конец соединения с сервером

  - HTTP Redirect — ваш перессылает на страницу с предупреждением об блокировке



# Атака DPI, десинхронизация и прочие синонимы

Целью атаки является сделать начало соединения (TLS Handshake - TLS Рукопожатие) незаметным для DPI и установить безопасное соединение (HTTPS) с сайтом.
После DPI не сможет прервать соединение
