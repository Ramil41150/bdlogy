# Устройство интернета (малосодержащий пересказ Википедии)

   Интернет представляет собой огромную сеть устройств, но без создания алгоритмов общения (то, как будут взаимодействовать пользователи) толку от него нет.
  
   ## Алгоритмы общения 
   Алгоритмы взаимодействия называют "протоколы".

## Модель OSI

 Для краткости описания всего интернета существует модель OSI.

 У модели OSI есть "уровни" (Layers). Для краткости Ln (n — уровень).

 Будут рассмотрены только L3, L4, L7.

  * L3 — сетевой уровень. Передача данных в одной части сети.

    Примеры: IP-адресация, маршрутизаторы.

  * L4 — транспортный уровень. Начиная с него берутся протоколы передачи данных.
    
    Например: UDP, TCP.

 * L7 — прикладной уровень. Общение со всем интернетом

  Например: WWW, HTTP, DNS

## Протоколы общения

   Есть 2 популярных протокола, которыми мы пользуемся постоянно: TCP и UDP.

   Эти протоколы используются для разных целей:

   ### Приемущества UDP и TCP в разных сферах
  
  #### TCP (Подходит для сайтов с текстовым содержимым)
  TCP - Transmission Control Protocol — протокол управления передачей.

 TCP/IP - популярная реализация, где используется TCP. Она имеет гарантии доставки и целостности данных, которая реализуется за счёт проверок пакетов переотправляя потерянные пакеты при обнаружении пропажи, а также шифрует ваши данные принудительно.

  # TCP - устройство

TCP является важной частью интернета.

 Пакеты (далее сегменты) в TCP имеют несколько частей.

  ## Флаги TCP
  
  (Флагов много, я опишу те, что задействуются в бд)
  
  * SYN - уточнение последовательности сегментов и их параметров
    
  * URG - выделение отдельного канала связи для передачи важных данных с высокой важностью

  * FIN — закрытие соединения с сервером

  * RST — мгновенный сброс соединения и отчистка данных о сессии

  ## Структура сегмента TCP
  У TCP есть заголовки, они содержат информацию о пакетах и в них множество информации для точной работы

   1. Порт источника (Source Port) - это идентификатор клиента, с которого отправляется сегмент(0-15 бит)

   2. Порт назначения (Destination Port) - идентификация порта, на который был отправлен пакет (16-31 бит)

   3. Порядковый номер сегмента (Sequence number) - 32-ух битное значение для различения размера пакета. Если он отправляется во время соединения с сервером, то имеет число от 0 до 4.294.967.295. (32-64 бит)

   4. ACK - номер подтверждения сегмента (64-96 бит)

   5. Длинна заголовка (Data offset) - указывает смещения (отступа от начала сегмента до части с данными, чтобы сервер правлино воспринимал части) заголовка для точного определения сервером (96-99 бит)

   6. Резерв - зарезервированные 3-4 бита, без использования должны быть заполнены нулями (99-102)

   7. Флаги - биты для управления соежинением

   8. Размер окна (Window Size) - представляет собой кол-во байт, которое можно отправить серверу без подтверждения их (пригодно к использованию только после обмена SYN)

   9. Контрольная сумма (checksum) - набор символов для проверки целостности файлов. 
  
  Допустим у нас есть файл hi.md, разобьём на несколько частей для передачи, сумма части = ABC
    Мы передаём его. Сумма совпала у одного пакета -  он передан успешно, иначе - будет не принят и запрошен заново (128-143)

   10. Указатель важности (Urgent pointer) - часть сегмента в 16 битах, где можно передать информацию о важности сегмента, например, выделить отдельную полосу обмена данными для быстрой доставки одного сегмента с помощью флага URG. (144-160)

  11. Опции (Options) - могут использоваться для разных целей, взависимости, как его воспринимает сервер. (160)
    
  12. Данные, которые требуется передать

  #### UDP (быстрый обмен данными, например, звонки и медиа)
   UDP - User Datagram Protocol, Протокол Пользовательских Датаграм.

   В отличие от TCP, в UDP отсутствует шифрование (для его реализации есть адаптация TLS - DTLS), что делает его быстрее, но более открыт к чтению.

   UDP позволяет мгновенно обеспечивать несколько соединений. Как и в TCP здесь есть реализация портов

   # Структура UDP пакета
  
  1. Порт отправителя (Source port, исходный порт) - идентификация порта отправителя, если подуразумевается двухстороннего общения между клиентом и сервером. Если в пакете будет не предусмотренна такая схема общения, то портом будет выступать "0" (Не обязателен)

  2. Порт получателя (Destination port) - идентификация порта получателя. (Обязателен)

  3. Длинна датаграммы (Length) - датаграмма содержит в себе 65.507 байт, если будет использован IPv4 (хотя может быть и 65.535). (В отличие от TCP здесь указана длинна заголовка и данных).

  4. Проверочная сумма (Checksum) - проверка всего пакета. Если сумма будет обозначена, но будет отсутствовать, то будет заполнена нулями. (Не обязательна в IPv4)

  5. Данные (Data) - остальная часть пакета
