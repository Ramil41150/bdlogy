 # Стратегии
Стратегии - это группа/набор групп из аргументов

   ## Группы
   Группы - это наборы аргументов на определённые случаи (подозрения в блокировке, при поломке SSL, для определённого домена/IP и т.д). Их можно различать по флагу -A, он позволяет как раз наладить автоматизацию в стратегии.

 ### -A — автоматизация стратегии

-At — если ByeDPI обнаружил блокировку соединения, то применит эту группу

-Ar — если ByeDPI заметит подмену IP адреса (такое может происходить, если оператор уведомляет о блокировке), то применит эту группу

-As — если произойдёт поломка SSL, то ByeDPI применит эту групп

-An — выход из группы с ограничителем (устанавливайте в начале стратегии эти группы: -Ku -a5 -An -J:"127.0.0.1/24" -d3 -An -o2)

   ## Фрагментация и атака
  
### -s — обычная нарезка TCP-пакета
   Нарезка TCP-пакета на 2 части. 1 часть: 1-n байт, 2 часть: n-? байт (n — указанное число).

#### Доп. фрагментация 

Для флагов -s, -d, -o, -q, -f, -r применима дополнительная фрагментация

Как это работает:

 -s1:2:3 — в таком случае пакет будет разделён 3 раза: после 1 байта, после 2, после 3. Всего 4 куска.

Также мы можем дополнять флагами флаг

 -s1:2:3+sme, напрмер

Таких флагов несколько:

 +s — TCP-пакет будет разделён на смещении SNI в Client Hello соединения + указанные позиции

 +h — пакет будет разделён на смещении Host из HTTP пакета + указанные позиции

 +n — от начала пакета

  Доп. флаги:

  -e — в конце

  -m — по середине

### -r - нарезка CH
  В списке "Содержание TLS Record (TLS Запись) для ClientHello" описанны смещения.
  
  Суть изпользования: сделать Запись нечиталебльной для DPI (хотя взависимости от провайдера ТСПУ можеты склеивать дробёжку Записи)
  
  Если писать -r?+s, то разрез будет на ? байте после SNI в ClientHello

  ### -d — обратная отправка TCP-пакета
  Работает как -s, но 1 часть отправляется с TTL, равным одному, после чего отправиться 2 часть, после которой протокол востановит 1 часть

Порядок отправки: -d3 — 3-n, 1-3

 ### -o — вне полосная отправка данных (OOB — Out-Of-Band)
  Похож на -s, но к первой части в конец добавляется URG-флаг (1 байт)
  
  Порядок отправки: -o2 — 1-3 (4 — URG), 2-n
  
 ### -q —  OOB + Disorder
 Является смесью -o и -d.
 
 Порядок отправки: -q2 — 3 (байт URG), 4 - ?, 1 - 2.

 ### -f — сгенерированный пакет
 Метод атаки, при котором на DPI приходит ложная информация, которая заставляет ТСПУ пропустить пакет далее.
 Он схож с -d, но после 2 части вставляются ложные данные

 Порядок отправки: -f2 — 2-n настоящие, 1-2 ложные, 1-2 настоящие.

 Доставка фейка до сервера — не есть хорошо. Мы должны не дать фейку быть принятым сервером. Для этого есть несколько методов:

 1. -t — установка TTL для фейка

 2. -S — установить проверку пакетов сервером

 3. Ретрансмиссионный подход — с помощью -d вызвать перезапись, которая случится перед сервером

## Другое

  ### -m - визуальная смена версии TLS
  Меняет третий байт TLS Record (минорную версию TLS)
  
  -m1 - TLS 1.0
  
  -m2 - TLS 1.1
  
  -m3 - TLS 1.2

  ### -Y игнорировании SACK проверки
 
   ByeDPI заставит игнорировать SACK, тем самым заставит постоянно переотправлять пакеты, даже если они целы и не имеют дефектов.
   
   Может помочь, если сервер зажуёт FAKE пакет и будет воспринимать его за настоящий
  
## Работа с fake пакетом

### -n - SNI для Fake'а 
Мы можем дополнительно установить SNI (Server Name Indication - Индикатор Имени Сервера).
Допустим, fonts.google.com.
  
> -Ku -a3 -An -o2 -d3 -At,r -f-1+hme -d3 -t5 -n fonts.google.com

### -Q - CH (ClientHello) для Fake'а
  * -Qr - случайные данные (если указанны другие параметры, то они будут с приоритетом)
  * -Qo - использовать обычный CH
  * -Qm=? - размер FAKE пакета (? замените на нужное кол-во байтов)

 ### -l — информация для сгенерированного пакета

  При указании -l его данные будут использоваться для всех фейков в группе.

  ### -S — MD5 проверка
 > [!TIP]
 > Данный аргумент лучше использовать на Linux-подобных ОС

  ByeDPI просит сервер проверять пакеты на предмет FAKE

  Если сервер заметит такой пакет, то пакет будет проигнорирован

  ### -t — TTL для сгенерированных пакетов

Установка TTL для сгенерированных пакетов (и TCP и UDP)
